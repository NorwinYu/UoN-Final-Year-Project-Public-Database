In the setup instructions for CF and CFI circa 2017 or earlier, you had to declare JSR308 as an bash env variable in your bash_rc. CF has since made all of such paths relative in its scripts. CFI might be (there can still be some hard coded paths in opprop/CFI). DLJC hasn't been updated to follow this paradigm, but the setup scripts should put it in $JSR308/dljc. Feel free to file a PR to https://github.com/opprop/do-like-javac to update it to use relative paths.
